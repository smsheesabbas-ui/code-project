# Feature 5: Clarity Dashboard

## Overview

The Clarity Dashboard provides a **single, minimal, and actionable view** of the businessâ€™ financial health.

Goals:

---

# EPIC 1: Dashboard Overview & Key Metrics

## Objective

Present core financial KPIs in a **simple, readable layout** to provide instant clarity.

## Business Value

Users immediately understand their cash position, revenue trends, and spending patterns without digging through reports.

## Scope

* Cash balance overview
* Revenue and expense summaries
* Top customers and suppliers
* Quick visual indicators for alerts and risks

## Functional Requirements

* Show current cash balance
* Display total revenue and expenses for selected period
* Highlight top 5 customers and suppliers
* Include trend arrows or percentages for month-over-month comparison
* Provide alert indicators for anomalies or cashflow risk

## Non-Functional Requirements

* Load in <2 seconds
* Mobile responsive
* Minimalist, no accounting jargon
* Color-coded alerts and trends

---

* Cashflow timeline
* Highlight anomalies or trends

## Functional Requirements

* Line chart for revenue vs expenses
* Bar chart for category breakdown

# EPIC 3: Customization & Interaction

## Objective

Allow users to **personalize dashboard view** and interact with insights for deeper clarity.

## Business Value

Users feel in control and can focus on metrics most relevant to their business.

## Scope

* Select visible metrics/cards
* Set alert thresholds
* Filter by date range or tags
* Drag & drop cards to reorganize

## Functional Requirements

* Toggle KPI visibility
* Adjust alert thresholds (e.g., low cash warning)
* Filter by tags, categories, or customers
* Rearrange dashboard layout

## Non-Functional Requirements

* Config changes persist per user
* Minimal UI clutter
* Mobile-friendly interaction
* Immediate feedback on changes

---

## Story 3.1: Select & Customize Metrics Displayed

### User Story

As a user,
I want to choose which metrics appear on my dashboard,
So I see only what matters to me.

### Acceptance Criteria

* Checkbox or toggle for each KPI
* Changes saved automatically
* Dashboard updates dynamically

### Edge Cases

* User hides all metrics
* Too many metrics selected
* Mobile layout adjustments

### Definition of Done

* Metrics selection works
* Changes persist across sessions
* Dashboard layout adjusts correctly

---

## Story 3.2: Filter by Date Range & Tags

### User Story

As a user,
I want to filter dashboard data by date or tags,
So I can focus on specific periods or categories.

### Acceptance Criteria

* Date range picker available
* Tag filter applied
* Charts and summaries update accordingly
* Filter resets easily

### Edge Cases

* No data in selected filter
* Multiple overlapping tags
* Invalid date ranges

### Definition of Done

* Filters functional
* Dashboard updates correctly
* Reset works

---

## Story 3.3: Drag & Reorder Dashboard Cards

### User Story

As a user,
I want to rearrange cards and panels,
So I can prioritize the most important insights.

### Acceptance Criteria

* Drag-and-drop supported

### Definition of Done
